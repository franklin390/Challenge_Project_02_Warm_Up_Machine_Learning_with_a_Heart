1. Descrição geral do problema¶

As doenas cardacas so a principal causa de morte no mundo. Para aprender a prevenir doenas cardacas, precisamos primeiro aprender a detect-las com segurana.

Nosso conjunto de dados de um estudo de doenas cardacas que est aberto ao pblico h muitos anos. O estudo coleta vrias medidas sobre a sade do paciente e as estatsticas cardiovasculares e, claro, torna as identidades dos pacientes annimas.

Os dados so fornecidos como cortesia do Cleveland Heart Disease Database) atravs do repositrio UCI Machine Learning.

Prevenir doenas cardacas importante. Bons sistemas orientados a dados para prever doenas cardacas podem melhorar todo o processo de pesquisa e preveno, garantindo que mais pessoas possam ter uma vida saudvel.

Nos Estados Unidos, os Centros de Controle e Preveno de Doenas so um bom recurso para obter informaes sobre doenas cardacas. De acordo com o site:

  • Cerca de 610.000 pessoas morrem de doenas cardacas nos Estados Unidos a cada ano - isso 1 em cada 4 mortes.
  • As doenas cardacas so a principal causa de morte para homens e mulheres. Mais da metade das mortes por doenas cardacas em 2009 ocorreram em homens.
  • A doena cardaca coronria (DCC) o tipo mais comum de doena cardaca, matando mais de 370.000 pessoas anualmente.
  • Todos os anos, cerca de 735.000 americanos sofrem um ataque cardaco. Destes, 525.000 so um primeiro ataque cardaco e 210.000 ocorrem em pessoas que j tiveram um ataque cardaco.
  • As doenas cardacas so a principal causa de morte para pessoas da maioria das etnias dos Estados Unidos, incluindo afro-americanos, hispnicos e brancos. Para ndios americanos ou nativos do Alasca e asiticos ou ilhus do Pacfico, as doenas cardacas perdem apenas para o cncer.

Para obter mais informaes, consulte o site dos centros de controle e preveno de doenas.

Objetivo: prever a classe binria heart_disease_present, que representa se um paciente tem ou no uma doena cardaca utilizando para a avaliao de desempenho dos modelos preditivos a serem criados a mtrica Logloss).

3.4.1.2 Funções para calcular estatísticas¶

Criaremos uma funo para padronizar as estatsticas que calcularemos para cada uma das variveis a serem estudadas.

In [138]:
# Definindo um funo para gerar um dataframe com as estatsticas de uma varivel do dataset.

varStats <- function(col, data, target = 'heart_disease') {

    if(!is.factor(data[, col])) {                 

        # Calcula estatsticas de variveis numricas.
        
        data %>%
            group_by_at(target) %>%
            summarize(
                Min       = min(get(col)),
                Q1        = unname(quantile(get(col), probs = c(0.25))),
                Median    = median(get(col)),
                mean      = mean(get(col)),
                Q3        = unname(quantile(get(col), probs = c(0.75))),
                Max       = max(get(col)),
                Sd        = sd(get(col)),
                Sk        = skewness(get(col)),
                Ck        = kurtosis(get(col))
            )

    } else {                                      

        # Calcula estatsticas de variveis categricas.

        l = list()

        # Contabiliza o nmero de registros dentro de cada classe da varivel.

        l$variableClasses <- data %>%
            group_by_at(col) %>%
            summarise(
                count = n(),
                prop  = n() / nrow(data) * 100
            )
        
        # Contabiliza o nmero de registros dentro de cada classe da varivel e segmenta cada subconjunto a partir da varivel alvo.

        l$variableClassesGroupedByTargetVariable <- data %>%
            group_by_at(c(col, target)) %>%
            summarise(
                count = n(),
                prop  = n() / nrow(data) * 100
            )

        # Retorna a lista com as estatsticas computadas.

        l
    }
}

O coeficente de Assimetria (Skewness) indica como os dados esto distribudos e para interpretar seu resultado podemos olhar a tabela a seguir:

ndice de Assimetria Descrio
SK 0 Os dados so simtricos. Tanto a cauda do lado direito quanto a do lado esquerdo da funo densidade de probabilidade so iguais;
SK < 0 A assimetria negativa. A cauda do lado esquerdo da funo densidade de probabilidade maior que a do lado direito e;
SK > 0 A assimetria positiva. A cauda do lado direito da funo densidade de probabilidade maior que a do lado esquerdo.

O coeficiente de Curtose (Kurtosis) uma medida que caracteriza o achatamento da curva da funo de distribuio e para interpretar seu resultado podemos olhar a tabela a seguir:

ndice de Curtose Descrio
CK 0 A distribuio normal e chamada de Curtose Mesocrtica;
CK < 0 A Cauda mais leve que a normal. Para um coeficiente de Curtose negativo, tem-se uma Curtose Platicrtica e;
CK > 0 A Cauda mais pesada que a normal. Para um coeficiente de Curtose positivo, tem-se uma Curtose Leptocrtica.

Ateno: H diferentes frmulas para calcular estes coeficientes. Mas, para este estudo utilizamos as funes fornecidas pela biblioteca e1071 com suas respectivas configuraes padro. Em caso de dvida, consulte a documentao.

5.2 Avaliando a importância das variáveis com o algoritmo Random Forest¶

Antes de iniciarmos a criao dos modelos propriamente dita, interessante definirmos o quanto cada varivel dentro do dataset ajuda a prever o valor da varivel target.

Para fazermos isto, utilizaremos o algoritmo Random Forest para computar tais valores. Como no sabemos qual o ajuste de parmetros do algoritmo mais adequado para o nosso conjunto de dados, criaremos modelos que possuam de 1 a 100 rvores com 1 a 100 ns.

Salvaremos a acurrica de cada modelo em um Dataframe. Note que por ser um processo de treinamento que tende a demandar muito tempo para ser processado, salvaremos os resultados obtidos em um arquivo CSV que pode ser carregado com todos os resultados gerados pelo bloco de cdigo a seguir.

In [ ]:
# Criando o dataframe para salvar os resultados dos modelos.

featuresRF <- data.frame()

# Definindo o nmero de ns e rvores a serem combinados para a criao de diferentes modelos.

nTrees <- 1:100
nNodes <- 1:100

# Define o nmero total de modelos a serem criados.

total <- length(nTrees) * length(nNodes)

# Define uma varavel auxiliar para permitir o acompanhamento do progresso na avaliao dos modelos criados.

count <- 0

for(t in nTrees) {
    
  for(n in nNodes) {
    
    # Define um seed para permitir que os mesmos resultados dos experimentos sejam reproduzveis.
    
    set.seed(100)
    
    # Cria o modelo Random Forest a ser avaliado.

    model <- randomForest(heart_disease ~ .,
                          data       = train, 
                          ntree      = t, 
                          nodesize   = n, 
                          importance = T)
    
    # Computa a Confusion Matrix gerada a partir do modelo criado.

    cm <- confusionMatrix(table(
        data      = model$y, 
        reference = model$predicted
    ))
    
    # Armazena os parmetros utilizados para criao do modelo e a acurcia obtida no dataframe.

    featuresRF <- rbind(featuresRF, data.frame(
        nodes    = n, 
        nTree    = t, 
        accuracy = unname(cm$overall['Accuracy'])
    ))
    
    # Incrementa o nmero de modelos avaliados.

    count <- count + 1

    # Imprime a porcetagem de progresso do treinamento e a melhor acurcia j alcanada.

    print(paste(100 * count / total, '%, best accuracy: ', max(featuresRF$accuracy)))
  }
}

# Salvando dataframe com os resultados gerados em um arquivo .csv.

write.csv(featuresRF, 'content/outputs/featuresRF.csv')
